Security & trust

Your data never leaves your infrastructure

ONDB is designed with a zero-trust architecture. You control access, encryption, and who can query.

Data stays in your infra

Connectors run in your environment. Queries are processed locally. Only responses leave your network.

End-to-end encryption

All data in transit is encrypted via TLS 1.3. Optional encryption at rest for cached responses.

Granular access controls

Define who can query what. Set rate limits, allowlists, and per-agent permissions from your dashboard.

Audit logging

Every query is logged with agent identity, timestamp, and payment details. Full compliance trail built in.

For agent builders

One SDK, every data source

Access private data across providers with a single query. Cross-provider joins and automatic revenue splits built in.

# Install
$ npm install @ondb/sdk
import { ONDB } from '@ondb/sdk';

const db = new ONDB({ apiKey: '...' });

// Query a provider's data (pay per query via x402)
const flights = await db.query('flights@travelco', { from: 'SFO' });

// Cross-provider join - revenue splits automatically
const deals = await db.query('flights@travelco + hotels@stayapp', {
    from: 'SFO', dates: '2025-03-01..07'
});

$ pip install ondb

No migration required. Our open-source connectors plug into your existing databases and APIs. Wrap any REST or GraphQL endpoint in minutes. Our universal query layer handles the rest, so your data stays where it is.

Data providers set prices per query. When an agent queries data, payment is initiated via x402. No invoices, no manual transfers. Top up with a card, cash out to your bank.

When someone queries your data, you earn. Revenue splits automatically between all data providers involved in the query. No negotiations, no manual tracking.
